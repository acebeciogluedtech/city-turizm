City Turizm web platformu, Türkiye merkezli bir turizm şirketinin kurumsal kimliğini, hizmetlerini ve kariyer fırsatlarını dijital ortamda temsil etmek üzere sıfırdan inşa edilmiş tam yığın (full-stack) bir web uygulamasıdır. Temel Özellikler

🌐 Çok Dilli Destek

Türkçe / İngilizce dil değiştirme; tüm sayfa içerikleri TR/EN olarak CMS'ten yönetilir. useLanguage() hook ile global dil durumu yönetimi.

✏️ Canlı İçerik Editörü

Admin panelinde her sayfanın içeriği bölüm bazlı düzenlenebilir. Değişiklikler Supabase'e yazılır, ISR cache temizlenerek anında siteye yansır.

📋 Başvuru Sistemi

Genel, Sürücü, Rehber ve Araç olmak üzere 4 farklı başvuru formu. PDF/görsel yükleme, Supabase Storage entegrasyonu ve admin bildirim sistemi.

Admin dashboard'unda mesaj, teklif, başvuru ve bülten aboneliklerini gerçek zamanlı takip eden badge ve popup bildirim sistemi.

🔍 SEO Yönetimi

Her sayfa için title, description, og:image, robots direktifleri admin panelinden düzenlenebilir. Veriler Supabase seo_settings tablosunda tutulur.

📱 Tam Responsive

Tailwind CSS v4 ile mobil öncelikli tasarım. Admin paneli mobil için kayan drawer sidebar, site sayfaları breakpoint bazlı grid sistem.

🔑 Ortam Değişkenleri

Mimari Kararlar

  • App Router kullanımı: Next.js 16 App Router, (site) route grubu ile admin ve kullanıcı rotaları birbirinden izole edilmiştir
  • Client / Server ayrımı: Statik içerikler Server Component olarak render edilir; interaktif bileşenler 'use client' ile işaretlenir
  • Supabase Admin Client: API route'larında service role key ile supabaseAdmin kullanılır; public client tarafında yalnızca anon key kullanılır
  • ISR (Incremental Static Regeneration): Sayfa içerikleri değiştiğinde revalidateTag ile cache temizlenerek statik sayfalar yeniden oluşturulur
  • Turbopack: Build süreci Turbopack ile hızlandırılmıştır; geliştirme ortamında hot-reload süresi minimize edilmiştir

Bölüm 5

API Katmanı

🔌 REST API

Tüm API endpoint'leri Next.js Route Handlers (app/api/) olarak implement edilmiştir. Server-side'da supabaseAdmin (service role) kullanılır; kimlik doğrulama sessionStorage tabanlı token kontrolü ile admin layout tarafında sağlanır.

İçerik ve Medya

GET
/api/admin/content?pageId={'{slug}'}
Belirtilen sayfa için tüm page_content kayıtlarını döner. pageId verilmezse tüm içerik gelir.
POST
/api/admin/content
Body: {'{pageId, sectionId, fieldId, tr, en}'} — İçerik kaydı oluşturur veya günceller (upsert). Başarıda revalidateTag çağrılır.
POST
/api/admin/upload
multipart/form-data — Dosyayı Supabase Storage'a yükler, public URL döner. PDF, görsel ve belge tiplerini destekler.
POST
/api/admin/convert-webp
Yüklenen görsel dosyasını WebP ve AVIF formatlarına dönüştürür, optimize edilmiş versiyonları döner.

Mesajlar ve Başvurular

GET
/api/admin/messages
contact_messages tablosundan tüm mesajları döner (created_at DESC).
PATCH
/api/admin/messages

/api/admin/applications
applications tablosundan tüm başvuruları döner. form_type'a göre filtreleme desteklenir.
PATCH
/api/admin/applications

Kimlik Doğrulama Akışı

Session-based Auth

Giriş formundan alınan bilgiler /api/admin/auth'a gönderilir. Başarılı doğrulama sonrası sessionStorage'a admin_auth ve admin_token anahtarları yazılır. AdminLayout her render'da bu değerleri kontrol eder; yoksa router.replace('/admin') çağrılır. Çıkış işlemi tüm session anahtarlarını temizler.

Bölüm 8

İçerik Yönetim Sistemi (CMS)

✏️ Live Editor

Platform, harici bir CMS gerektirmeden kendi bünyesinde barındırdığı canlı editör altyapısına sahiptir. Her sayfa içeriği page_content tablosunda page_id · section_id · field_id üçlüsü ile anahtar-değer şeklinde saklanır.

Veri Akışı

1. Sayfa yüklendiğinde (Server Component):
   fetchPageContent(pageId) → Supabase → BilingualContent map

2. Client Component'te içerik okuma:
   function v(sectionId, fieldId, fallback):
     content[`${sectionId}.${fieldId}`]?.[lang] ?? fallback

3. Admin editörde canlı düzenleme:
   Alan seçilir → sağ panel açılır → input değişir
   → handleLiveChange({ sectionId, fieldId, tr, en })
   → liveEdit state → g() fonksiyonu önce liveEdit'i kontrol eder
   → görsel anlık güncellenir (DB yazılmadan)

4. Kaydet butonu:
   POST /api/admin/content → Supabase upsert
   → revalidateTag('page-content')
   → ISR cache temizlenir → site sayfası yeniden render

GetVal Tip Sistemi

İçerik Okuma Fonksiyonları

Ana editör bileşeninde g(sectionId, fieldId, fallback?) iki argümanlı bir fonksiyon olarak tanımlıdır. Preview alt bileşenlerine GetVal = (fieldId: string) => string tipinde curried fonksiyon geçilir. Bu sayede alt bileşenler section context'ini prop olarak taşımak zorunda kalmaz. Footer ve Navbar özel editörleri kendi section helper'larını (br, ct, nv…) bu curried pattern ile oluşturur.

📄 Textarea

Çok satırlı metin. Açıklamalar, paragraflar.

</form>
⚠️

Önemli: Tüm formlarda noValidate attribute zorunludur. Browser native validation, required alanlar boşken formu submit etmeden sessizce engeller; bu durumda onSubmit tetiklenmez ve spinner asla gösterilmez. noValidate + JS guard pattern bu sorunu çözer.

Dosya Yükleme Akışı

1. Kullanıcı dosya seçer

Input onChange ile File nesnesi state'e alınır. Dosya tipi ve boyut kontrolü yapılır.

2. Form submit edilir, spinner başlar

setSubmitting(true) çağrılır, buton disabled + "Yükleniyor..." mesajı + CSS animate-spin gösterilir.

3. Dosya Storage'a yüklenir

FormData ile POST /api/admin/upload → Supabase Storage bucket → public URL döner.

4. Başvuru kaydedilir

Form verileri + dosya URL'si POST /api/admin/applications ile Supabase'e yazılır.

5. Spinner durur, başarı mesajı gösterilir

finally bloğunda setSubmitting(false), ardından success state ile teşekkür ekranı render edilir.

Deployment ve CI/CD

🚀 Vercel

Proje GitHub → Vercel entegrasyonu ile sürekli deployment pipeline'ına sahiptir. main branch'e her push sonrası Vercel otomatik build başlatır.

Build Süreci

1. Kod Push

git push origin main → GitHub acebeciogluedtech/city-turizm repository'sine yazılır.

2. Vercel Build Tetiklenir

Vercel webhook GitHub'dan build isteği alır. Washington DC (iad1) bölgesinde Turbo Build Machine (30 core, 60 GB) devreye girer.

3. npm install

384 paket yüklenir (~8 saniye).

4. next build (Turbopack)

TypeScript derleme (~7 saniye), statik sayfa üretimi (58 route, 29 worker paralel), optimize edilmiş chunks.

5. Deployment

Build başarılı olursa production domain'ine otomatik yayınlanır.

Ortam Değişkenleri

DeğişkenKullanım YeriAçıklama
NEXT_PUBLIC_SUPABASE_URLClient + ServerSupabase proje URL'si
NEXT_PUBLIC_SUPABASE_ANON_KEYClientPublic anon key (RLS korumalı)
SUPABASE_SERVICE_ROLE_KEYServer only (API routes)Tam yetki service role key — asla client'a açılmaz
⚠️

Güvenlik: SUPABASE_SERVICE_ROLE_KEY yalnızca server-side API route'larında (supabase-admin.ts) kullanılır. Bu key NEXT_PUBLIC_ prefix'i olmadığından client bundle'a dahil edilmez. RLS politikaları, public API ile yapılabilecek işlemleri sınırlandırır.

Build Gereksinimleri

  • TypeScript strict mode: Tüm tip hataları build'i durdurur; any implicit kullanımı yasaktır
  • Suspense boundary: useSearchParams() kullanan her client bileşen <Suspense> ile sarılmalıdır
  • Framer Motion ease tipleri: Bezier eğrisi dizileri [number, number, number, number] tuple olarak cast edilmelidir
  • Motion prop'ları: Yalnızca resmi Framer Motion prop'ları kullanılabilir; initialOpacity gibi non-standard prop'lar derleme hatasına neden olur
